# When Does the Body Matter?

Code, data, and supplementary materials for the preprint:

**Lucken, C.** When Does the Body Matter? A Dynamical Signature of Coupling Dependence in Evolved Neural Controllers. Submitted to *SAB 2026: Simulation of Adaptive Behavior — From Animals to Animats*.

## What's here

This repository contains everything needed to reproduce the analyses reported in the paper, plus the simulation framework the experiments were built on.

- `paper/paper2_sab.pdf` — the submitted paper (pre-review).
- `simulation/` — the simulation framework (`ctrnn.py`, `evolutionary.py`, `microworld.py`, `analysis.py`) and the paper-specific scripts under `simulation/experiments/paper2/`.
- `data/` — JSON outputs of the canonical analyses (the numbers in the paper come from these files).
- `genotypes/` — evolved CTRNN parameters for three representative seeds per network size, in NumPy format.
- `figures/` — paper figures (PDF and PNG); see `figures/README.md`.
- `docs/` — extended supplementary analysis writeups.
- `requirements.txt` — Python dependencies (Python ≥ 3.8).

## What the paper does, in a paragraph

The paper proposes corrected versions of an experimental tool used in evolutionary robotics — *ghost conditions* — that make neural-state divergence a usable dependent variable. Applied across 60 evolved CTRNN controllers performing phototaxis, the corrected conditions reveal a dynamical signature of coupling dependence: networks with high perturbation growth rate, input-sensitive attractor landscapes, and (most strongly) high inter-trial trajectory variance (Spearman ρ = 0.80) depend on continuing sensorimotor coupling. Network size shifts the probability that evolution discovers this regime but does not determine it. The signature aligns with the edge-of-chaos regime in reservoir computing and with input-driven attractor structures in trained recurrent networks.

## Reproducing the results

Requires Python ≥ 3.8.

```bash
pip install -r requirements.txt
```

The 60-condition results were generated by the `phase_a_10seeds` driver in `simulation/experiments/paper2/`. Six network sizes (n = 2, 3, 4, 5, 6, 8), ten random seeds per size. Each condition evolves a CTRNN with the Microbial Genetic Algorithm (Harvey 2009), then evaluates the corrected ghost conditions (frozen body, disconnected, counterfactual) on the evolved network.

Run all commands from the repository root so the `simulation` package is importable.

### Quick check (under two minutes)

```bash
python -c "from simulation.experiments.paper2.constitutive_vs_causal import run_proof_of_concept; print(run_proof_of_concept()['embodiment_analysis']['constitutive_score'])"
```

This evolves one small network and runs it through the three corrected ghost interventions, printing the composite ED score. (The variable is named `constitutive_score` for historical reasons; it implements the ED score described in §3.3 of the paper.)

### Reproduce the paper's analyses against the existing data

```bash
python -m simulation.experiments.paper2.dynamical_analysis_60     # dynamical-regime predictors
python -m simulation.experiments.paper2.attractor_geometry         # attractor-landscape probing
python -m simulation.experiments.paper2.mechanistic_analysis       # weight-level and partial-correlation analyses
python -m simulation.experiments.paper2.statistical_corrections    # Benjamini-Hochberg FDR table
python -m simulation.experiments.paper2.generate_figures           # regenerate paper figures
```

### Re-run the underlying simulation

Slow — about two hours on a single core.

```bash
python -m simulation.experiments.paper2.phase_a_10seeds
```

Outputs land in the same JSON format used in `data/`. Results are deterministic given the seed schedule (42, 137, 256, plus seven additional seeds per size for the full 60-condition design).

## Data dictionary

- `data/phase_a_10seeds_with_all_genotypes.json` — the canonical 60-condition file. Includes evolved genotype, ghost-condition divergences (FB, DC, CF, raw and capped), and composite ED scores. The numbers in the paper come from this file.
- `data/dynamical_analysis_60.json` — perturbation growth rate, state entropy, inter-trial trajectory distance, fraction amplifying, participation ratio, maximum Lyapunov exponent, all computed on the evolved embodied trajectories.
- `data/attractor_geometry_60.json` — fixed-point locations, attractor classifications, bifurcation counts, input-sensitivity scores across the input-amplitude scan.
- `data/mechanistic_analysis.json` — weight-level analyses (self-connection polarity, maximum real eigenvalue) and partial-correlation results.
- `data/statistical_corrections.json` — full 26-test Benjamini-Hochberg FDR table referenced in §4 of the paper.
